## Summary

Replace BLIP-base with Florence-2-base for multi-task AI frame analysis, add optional CLIP ViT-B/32
semantic embeddings, wire in optional Claude Haiku LLM executive summaries, overhaul the Intelligence
Report with a format modal / 4-stage SSE progress / Scene Breakdown section / SVG activity timeline,
and polish Processing-page logs and the debug drawer.

**Language/Version**: Python 3.12; JavaScript (ES2022 vanilla)
**Primary Dependencies**: FastAPI 0.111, PyQt6 6.7, transformers (already installed), torch (transitive), Pillow, numpy; optional: open-clip-torch, anthropic
**Storage**: No new persistent storage. CLIP embeddings are output artifacts (`.clip.npy` sidecars next to existing thumbnails). `localStorage` persists report format choice (user-config exemption, Principle I).
**Testing**: pytest 8.2 + pytest-asyncio; frontend JS covered by quickstart.md manual scenarios (constitution Principle III frontend exemption)
**Target Platform**: Windows 11 desktop app (PyQt6 shell); web frontend via QWebEngineView
**Project Type**: Desktop app (PyQt6 wrapping FastAPI + vanilla JS SPA)
**Performance Goals**: Report generation completes in <60 s for 12-event run on CPU; SSE progress events arrive within 500 ms of stage change

**Scale/Scope**: One job at a time; typical detection run 5–30 events; report covers at most 30 thumbnails.

## Architecture

### AI Model Stack

```
Florence-2-base  (transformers, already installed)
   ↓ <MORE_DETAILED_CAPTION>   → 2–5 sentence paragraph per event
   ↓ <OD>                      → detected objects + bounding boxes
   ↓ <REGION_CAPTION>          → crop description for each detected object
   → structured FrameAnalysis dict stored in event session data

CLIP ViT-B-32-quickgelu  (open-clip-torch, optional)
   → 512-dim L2-normalised embedding per thumbnail
   → written to {thumb_stem}.clip.npy alongside thumbnail
   → no runtime impact when open-clip-torch absent

Claude Haiku API  (anthropic, optional, ANTHROPIC_API_KEY env var)
   → receives structured event JSON as context
   → produces analyst-quality executive summary prose
   → falls back to enriched rule-based NarrativeSynthesizer on any failure
```

**Fallback chain** (executive summary):
```
1. Claude Haiku API available + ANTHROPIC_API_KEY set    → LLM prose
2. Florence-2 available (enriched data)                  → enriched rule-based
3. Baseline (no AI library)                              → plain NarrativeSynthesizer
```

### SSE Stage Progress Pattern

The POST `/job/intel-report/export` endpoint runs in FastAPI's sync thread pool
(synchronous handler → FastAPI uses `anyio.to_thread.run_sync` under the hood).
The async SSE endpoint `/api/stream` continues polling `session.snapshot()` while
the POST runs in its thread. Stage progress is written to session state using
`session.update(...)` at each stage boundary; the SSE stream picks it up within
the next 500 ms poll interval.

**New session fields** (added to `app/session.py` `_DEFAULTS`):
```python
"report_stage": "",           # "" | "thumbnails" | "ai_analysis" | "markdown" | "pdf"
"report_stage_current": 0,    # current item index
"report_stage_total": 0,      # total item count
"report_stage_timestamp": "", # source video timestamp e.g. "00:01:23"
```

**New SSE event types** emitted by `stream.py` when `report_stage != ""`:
```json
{"type": "report_stage", "stage": "ai_analysis", "current": 5, "total": 12, "ts": "00:00:45"}
{"type": "report_done",  "md_path": "...",        "pdf_path": "..."}
```

### Report Format Modal Flow

```
[Export page loads]
  → fetch /api/job/status
  → show "Generate Intelligence Report" button (enabled if events > 0)

[User clicks button]
  → show format modal (checkboxes pre-selected from localStorage)
  → user checks: md=true pdf=true / md=true pdf=false / md=false pdf=true
  → user clicks "Generate →"

[Frontend calls POST /api/job/intel-report/export]
  → body: {"formats": ["md", "pdf"]} or {"formats": ["md"]} or {"formats": ["pdf"]}
  → opens SSE listener for report_stage events
  → shows 4-stage progress UI (replaces existing spinner)

[Backend generates, emits stage events, returns]
  → report_done event → close SSE listener
  → show success card with file links
```

### Scene Breakdown Section

Top 5 events by confidence score, each rendered as a card:
- Annotated thumbnail (bounding boxes drawn with PIL `ImageDraw` at report time)
- Full paragraph from Florence-2 `<MORE_DETAILED_CAPTION>`
- Italic sub-line from `<REGION_CAPTION>` (if YOLO detections present)
- Detection labels with confidence

Bounding box annotation is computed during `intel_report_renderer.py` rendering,
not stored to disk. The annotated version is base64-encoded inline into HTML.

### SVG Activity Timeline

Static inline SVG, generated by `intel_report_renderer.py`. Renders identically
in HTML preview and Qt PDF (no JS required, no interactive behaviour).

```
viewBox="0 0 800 48"
Background rail: rect x=0 y=20 w=800 h=8 fill=#2e3147 rx=4
Per-event tick: rect x=(t/dur*800) y=(20-h/2) w=4 h=(conf*32) fill={colour} rx=2
  where h = max(8, confidence * 32)
Colour coding:
  MOG2 detection  → #6b7280 (grey)
  YOLO person     → #3b82f6 (blue)
  YOLO vehicle    → #f97316 (orange)
  YOLO other      → #8b5cf6 (purple)
```

### Processing Page Log Panel

Togglable panel (hidden by default). Show/Hide button in page header.

```
── Starting detection ─────────────────────────
14:32:01  INFO   Loading YOLO model…
14:32:03  INFO   Model ready (2.1s)
── Events found ────────────────────────────────
14:32:45  EVENT  Person @ 00:00:45  conf 87%
14:33:02  WARN   Low confidence event skipped
```

Severity colour mapping:
- `INFO` → grey (`#9ca3af`)
- `EVENT` → blue (`#3b82f6`)
- `WARN` → amber (`#f59e0b`)
- `ERROR` → red (`#ef4444`)

Copy button: copies all log lines as plain text to clipboard.

### Debug Drawer Enhancements

Each network request entry in `debug-log.js` gains:
- Timestamp prefix: `HH:MM:SS.mmm`
- Fetch duration: `(123 ms)` appended
- HTTP status badge: `200 OK` or `404 Not Found`
- Error left border: `border-left: 3px solid #ef4444` when status >= 400
- Count badge on toggle button: total request count displayed as a chip

## Global Constraints (for SDD implementers)

These constraints apply to ALL tasks and are non-negotiable:

1. **All file paths via `pathlib.Path`** — no bare string concatenation
2. **No `frame_describer.py` imports after T003** — all callers use `frame_analyzer`
3. **Graceful optional imports** — `open-clip-torch` and `anthropic` must use `try/except ImportError`; no `ImportError` must reach the user
4. **Session state via `session.update()`** — never mutate the live dict directly
5. **TDD sequence** — write test → confirm fail → implement → confirm pass → commit; no exceptions for `app/core/` and `app/api/` files
6. **`FrameAnalysis` dict shape** — all callers must handle the exact shape: `{caption, object_caption, detections, clip_embedding_path}` where detections is `[{label, bbox}]` and clip_embedding_path is `str | None`
7. **SSE event backwards compatibility** — new `report_stage` and `report_done` event types are additive; existing frontend consumers must not break
8. **Report format param** — `formats` body param on the export endpoint must default to `["md", "pdf"]` for backwards compatibility
