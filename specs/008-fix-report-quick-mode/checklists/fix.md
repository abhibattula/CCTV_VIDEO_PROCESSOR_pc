# Bug Fix Requirements Quality Checklist: Phase 8

**Purpose**: Validate requirement quality for the 4 bug fixes (Florence-2 timeout, SSE socket exception, thumbnail progress, img guard)
**Created**: 2026-06-29
**Feature**: [spec.md](../spec.md)

## Requirement Completeness

- [ ] CHK001 - Is the exact threshold for the per-task timeout (90 s) specified as a hard cut-off, or could it be interpreted as a soft warning? [Clarity, Spec §FR-001]
- [ ] CHK002 - Is the maximum token count (64) specified as the value for all three Florence-2 `model.generate()` call sites, or only the caption task? [Completeness, Spec §FR-001]
- [ ] CHK003 - Are the fallback contents defined when Florence-2 times out on an event? The spec says "rule-based synthesis text" but does not specify which NarrativeSynthesizer function provides it or where it appears in the report. [Completeness, Spec §FR-002]
- [ ] CHK004 - Is the thumbnail progress requirement scoped only to the thumbnails SSE stage, or does it also cover the percent display in the browser? [Clarity, Spec §FR-003]
- [ ] CHK005 - Does the SSE disconnect requirement specify what "caught gracefully" means — silent (no log at all), debug-level log, or warning-level log? [Clarity, Spec §FR-004]
- [ ] CHK006 - Is the `annotated_thumb_b64` guard requirement scoped to the Scene Breakdown section only, or does it apply to all `<img>` tags that use base64 data in the template? [Completeness, Spec §FR-005]

## Requirement Clarity

- [ ] CHK007 - Is "the report still completes" (Spec §US1 Scenario 2) sufficiently defined — does it mean the API returns HTTP 200 with a valid HTML body, or just that no exception is raised? [Clarity, Spec §US1]
- [ ] CHK008 - Is "no unhandled exception traceback" (Spec §US2 Scenario 2) measurable? The criterion relies on terminal output inspection; is there a programmatic assertion defined? [Measurability, Spec §SC-004]
- [ ] CHK009 - Does SC-003 ("Thumbnails SSE stage shows 0% at start and advances to 100% only after thumbnail files are confirmed written") define what "confirmed written" means — return of the function, or explicit file-existence check? [Clarity, Spec §SC-003]

## Requirement Consistency

- [ ] CHK010 - Is SC-001 (< 9 min for 5 events) consistent with FR-001 (< 90 s per event)? The math is 90 s × 3 tasks × 5 events = 22.5 min, not 9 min. The spec uses "2 safety margin" but 90 × 3 × 2 = 9 min only if there is 1 event × 3 tasks × 2 margin. [Consistency, Spec §SC-001 vs FR-001]
- [ ] CHK011 - Does FR-002 ("rule-based synthesis text") align with US1 Scenario 2 ("shows rule-based synthesis ... not blank")? Both describe the same behavior, but FR-002 says "rule-based synthesis text for that event" while US1 says "completes and shows rule-based synthesis". Are they referring to the same output location (per-event description vs executive summary)? [Consistency, Spec §FR-002 vs US1 Scenario 2]
- [ ] CHK012 - Is the SSE disconnect fix (FR-004) consistent with the report-continues requirement (US1 Scenario 3)? FR-004 specifies the server must not crash; US1 Scenario 3 requires generation to continue to completion. Are both behaviors specified in a single coherent requirement or split across incompatible places? [Consistency, Spec §FR-004 vs US1 Scenario 3]

## Acceptance Criteria Quality

- [ ] CHK013 - Is SC-002 ("at least one event has a non-empty AI caption") verifiable in an automated test, or does it require real Florence-2 inference (which is slow and may be skipped)? Is the test-time approach specified? [Measurability, Spec §SC-002]
- [ ] CHK014 - Is SC-006 ("all 128 existing tests continue to pass") sufficient as a regression gate, given that the thumbnail progress fix and SSE fix are tested by new tests? Are the new test names or coverage scope specified? [Completeness, Spec §SC-006]

## Edge Case Coverage

- [ ] CHK015 - Does the spec define behavior when Florence-2 times out on ALL events (not just some)? Is the executive summary still generated by rule-based synthesis, or is a specific error message shown? [Edge Case, Spec §Edge Cases]
- [ ] CHK016 - Does the spec define behavior when the SSE client reconnects after a disconnect mid-generation? Is there a requirement about duplicate events or state replay? [Edge Case, Spec §Edge Cases]
- [ ] CHK017 - Does the spec address the case where `thumbnail_gen.run()` itself raises an exception? Is there a requirement that thumbnail failures are caught without preventing the rest of the report? [Coverage, Gap]

## Notes

- CHK013 and CHK014 are test-scope questions that will affect task generation (skippable tests vs mandatory tests).
