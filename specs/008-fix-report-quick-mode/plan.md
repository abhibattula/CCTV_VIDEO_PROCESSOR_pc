# Implementation Plan: Phase 8 — Report Fix + Quick Mode UI

**Branch**: `008-fix-report-quick-mode` | **Date**: 2026-06-29 | **Spec**: [spec.md](spec.md)

## Summary

Fix four confirmed bugs that cause the AI Intelligence Report to produce empty descriptions and unstable progress (Florence-2 inference timeout, SSE socket exception, premature thumbnail progress, broken `<img>` tag). Add one UI button ("Quick Report PDF") that gives an immediate motion-only PDF without waiting for AI. All changes target 5 existing files; no new modules or dependencies.

## Technical Context

**Language/Version**: Python 3.12 + JavaScript (vanilla ES2022, no build step)  
**Primary Dependencies**: FastAPI 0.115, PyQt6 6.9, transformers 5.x, PyTorch 2.x, OpenCV, Jinja2  
**Storage**: In-memory session (app/session.py). Job artifacts (thumbnails, heatmap) written to temp job_dir.  
**Testing**: pytest (128 tests passing on branch 007-ui-ai-overhaul)  
**Target Platform**: Windows 11 desktop (offline), single-user  
**Project Type**: Desktop application — PyQt6 shell + embedded FastAPI web server  
**Performance Goals**: AI caption completes in < 90 s per event; Quick PDF in < 5 s  
**Constraints**: CPU-only, fully offline, no npm/build step, no new Python packages  
**Scale/Scope**: One job at a time, up to ~20 events per job
