### T003: app/api/presets.py — Implementation

Implemented the complete presets router per plan.md's specification:

**Features**:
- `_load()` helper: Returns [] if file missing/corrupted (never crashes on JSON parse errors)
- `_save()` helper: Creates parent dirs as needed, writes pretty-printed JSON
- `GET /api/presets`: Returns list of all presets
- `POST /api/presets`: Creates new preset with validation:
  - Rejects empty/whitespace-only names (400)
  - Rejects case-insensitive collisions with built-in names {"Security Report", "Evidence Pack", "Quick Highlights"} (400)
  - Rejects case-insensitive + trimmed collisions with existing custom presets (400)
  - On success, returns the created preset object
- `DELETE /api/presets/{name}`: Deletes preset by name, returns 404 if not found

**Key implementation details**:
- Case-insensitive comparison uses `.lower()` for both built-in and custom names
- Trimmed comparison via `name.strip()` before validation
- Pydantic model `PresetCreateRequest` for request validation
- All endpoints use `JSONResponse` for consistency
