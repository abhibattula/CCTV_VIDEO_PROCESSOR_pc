# Implementation Plan: AI Fix, Performance & Cross-Platform Support

**Branch**: `011-ai-fix-perf-platform` | **Date**: 2026-06-30 | **Spec**: [spec.md](spec.md)  
**Research**: [research.md](research.md) | **Data Model**: [data-model.md](data-model.md)

## Summary

Three-track stabilisation phase. Track 1 fixes Florence-2 producing raw token garbage in Intelligence Reports (three concurrent root causes) and restores the live log panel after SSE disconnects. Track 2 eliminates YOLO cold-start lag and UI freezes during long detection runs. Track 3 makes the full app cross-platform (macOS, Linux desktop, Linux headless, Raspberry Pi) with automatic AI disable on low-RAM devices.

**Key research findings that change the original design:**
- `LogBuffer.subscribe()` already replays 100 lines — no new `snapshot()` method needed; frontend SSE reconnect is the only fix required.
- `detection_engine.py` already has a `frame_skip` parameter — YOLO detector only needs the new feature.
- `max_new_tokens` raised to **100** (not 150) to stay safely within the 90s task timeout.
- Warning suppression applied **inside `_run_task()`** (not around `_run_in_daemon` call) to reach the daemon thread's warning state.

### Phase 2: Track 1 — Log Panel SSE Reconnect

**File**: `static/js/pages/processing.js` (frontend exemption — documented in quickstart.md)

Current `onerror` (line 197-200):
```js
evtSource.onerror = () => {
  if (evtSource) { evtSource.close(); evtSource = null; }
  addLogEntry("WARN", "SSE connection lost — switching to polling");
  startPolling();
};
```

Replace with a reconnect wrapper:
```js
let _sseRetries = 0;
const _SSE_MAX_RETRIES = 5;
const _SSE_BACKOFF_MS = 3000;

function connectSSE() {
  evtSource = new EventSource("/api/stream");
  evtSource.onmessage = /* ... existing handler unchanged ... */;
  evtSource.onerror = () => {
    if (evtSource) { evtSource.close(); evtSource = null; }
    if (_sseRetries < _SSE_MAX_RETRIES) {
      _sseRetries++;
      addLogEntry("WARN", `SSE lost — reconnecting (${_sseRetries}/${_SSE_MAX_RETRIES})…`);
      setTimeout(connectSSE, _SSE_BACKOFF_MS);
    } else {
      addLogEntry("ERROR", "SSE connection lost — switching to polling");
      startPolling();
    }
  };
}
connectSSE();
```
Reset `_sseRetries = 0` on successful message receipt (inside `onmessage`).

**Why no server-side change**: `LogBuffer.subscribe()` already replays the last 100 log lines into the new subscriber queue. On SSE reconnect, a new subscribe() call is made, which replays missed logs automatically.

### Phase 5: Track 3 — Platform Fixes

**New file**: `app/utils/platform.py`
```python
import os
import platform
from pathlib import Path

def get_desktop_path() -> str:
    system = platform.system()
    if system == "Windows":
        try:
            import ctypes, ctypes.wintypes
            buf = ctypes.create_unicode_buffer(ctypes.wintypes.MAX_PATH)
            ctypes.windll.shell32.SHGetFolderPathW(0, 0, 0, 0, buf)
            if buf.value:
                return buf.value
        except Exception:
            pass
        return str(Path.home() / "Desktop")
    elif system == "Darwin":
        return str(Path.home() / "Desktop")
    else:  # Linux, Pi
        xdg = os.environ.get("XDG_DESKTOP_DIR", "").strip()
        if xdg and Path(xdg).is_dir():
            return xdg
        desktop = Path.home() / "Desktop"
        if desktop.is_dir():
            return str(desktop)
        downloads = Path.home() / "Downloads"
        return str(downloads if downloads.is_dir() else Path.home())
```
