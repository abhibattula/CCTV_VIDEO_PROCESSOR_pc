## GET /api/job/intel-report.html

**Purpose**: Returns a self-contained HTML document for Qt's `printToPdf` to consume.  
**Caller**: `shell/main_window.py` → `_generate_intel_report_pdf()` via hidden `QWebEnginePage`

### Request

No body, no query parameters.

### Guards (evaluated in order)

| Condition | Response |
|-----------|----------|
| No active job (`not snap.get("job_id")`) | `400 {"error": "No active job"}` |
| Status is `"detecting"` | `400 {"error": "Detection is still in progress for this job"}` |
| No included events | `400 {"error": "No events to report — no events are currently included"}` |

### Response (success)

- **Status**: `200 OK`
- **Content-Type**: `text/html; charset=utf-8`
- **Body**: Self-contained HTML string from `intel_report_renderer.render(context)`
  - All images embedded as `data:image/jpeg;base64,...` URIs
  - All CSS inline (`<style>` block in `<head>`)
  - No external resource references (fonts, scripts, or images via URL)

### Side Effects

1. Calls `thumbnail_gen.run()` (idempotent) to ensure thumbnails exist
2. Base64-encodes thumbnails and heatmap for embedding
3. Calls `FrameDescriber.describe()` for each included event thumbnail (may block for ~2-3s/event on CPU on first use, or ~1-2 min if model download is triggered)

### Error Responses

| Condition | Status | Body |
|-----------|--------|------|
| Guard fails | 400 | `{"error": "..."}` |
| File I/O error (thumbnail/heatmap) | 400 | `{"error": "Could not access a required file: ..."}` |

## POST /api/job/intel-report/export

**Purpose**: Writes the Markdown intelligence report file to `output_dir` and returns the path.  
**Caller**: `static/js/pages/export.js` → `fetch('/api/job/intel-report/export', {method: 'POST'})`

### Request

No body required. (Reserved for future `output_dir` override parameter, following the existing `EventLogExportRequest` pattern.)

### Guards (evaluated in order)

| Condition | Response |
|-----------|----------|
| No active job (`not snap.get("job_id")`) | `400 {"error": "No active job"}` |
| Status is `"detecting"` | `400 {"error": "Detection is still in progress for this job"}` |
| No included events | `400 {"error": "No events to report — no events are currently included"}` |
| `output_dir` is inaccessible | `400 {"error": "Could not access output folder: ..."}` |

### Response (success)

```json
{
  "md_path": "/absolute/path/to/{stem}_intelligence_{YYYYMMDD_HHMMSS}.md",
  "moondream_available": true
}
```

| Field | Type | Notes |
|-------|------|-------|
| `md_path` | `string` | Absolute path to the written Markdown file |
| `moondream_available` | `boolean` | `FrameDescriber.is_available()` — frontend uses to show/hide install notice |

### Side Effects

1. Calls `thumbnail_gen.run()` (idempotent) to ensure thumbnails exist
2. Calls `FrameDescriber.describe()` for each included event
3. Calls `narrative_synthesizer.*` functions to build context
4. Writes `{stem}_intelligence_{YYYYMMDD_HHMMSS}.md` to `output_dir`
5. Returns path — PDF generation is triggered separately by the frontend via Qt event

### Output File Naming

```
{source_stem}_intelligence_{YYYYMMDD_HHMMSS}.md
{source_stem}_intelligence_{YYYYMMDD_HHMMSS}.pdf   ← written by Qt after frontend receives md_path
```

Both files are written to the same `output_dir`. The timestamp in the filename is the same
for both (taken once at the start of the POST request, passed to Qt via the JS event payload).

### Error Responses

| Condition | Status | Body |
|-----------|--------|------|
| Guard fails | 400 | `{"error": "..."}` |
| File I/O error | 400 | `{"error": "Could not access output folder: ..."}` |

## Frontend → Qt Event Flow (Intel Report PDF)

The PDF is not returned by the API — it is generated by the Qt shell using `printToPdf`:

```
User clicks "Generate Intelligence Report"
  → JS: POST /api/job/intel-report/export
  → API: writes Markdown, returns {md_path, moondream_available}
  → JS: reads md_path, derives pdf_path (same stem + .pdf)
  → JS: dispatchEvent(new CustomEvent('cctv:generate-intel-report', {detail: {pdf_path}}))
  → Qt: _handle_browse_flags() detects window._cctvGenerateIntelReport == true
  → Qt: _generate_intel_report_pdf(pdf_path)
      → hidden QWebEnginePage loads GET /api/job/intel-report.html
      → printToPdf(pdf_path) on loadFinished
      → deleteLater() + list removal on pdfPrintingFinished
```

This is identical to Phase 5's `cctv:generate-pdf-report` / `_generate_pdf_report()` flow.
