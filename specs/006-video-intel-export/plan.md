# Implementation Plan: Video Intelligence Export

**Branch**: `006-video-intel-export` | **Date**: 2026-06-25 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/006-video-intel-export/spec.md`

## Summary

Generate a "Video Intelligence Report" (Markdown + PDF) from any completed or cancelled
detection run. The report synthesises structured event data into natural language,
optionally enriching each event with a visual description from the local Moondream2 vision
model. The Markdown output is structured to serve as RAG context for Phase 7's in-app
chatbot.

Architecture pattern: two new API endpoints in `app/api/job.py` (GET html for Qt
printToPdf, POST export for Markdown file write), three new `app/core/` modules
(frame_describer, narrative_synthesizer, intel_report_renderer), one new Jinja2 template,
one new test file, and frontend additions to `static/js/pages/export.js` and
`shell/main_window.py`. All patterns mirror Phase 5's existing incident report exactly.

## Technical Context

**Language/Version**: Python 3.11 (backend), JavaScript ES2023 modules (frontend)  
**Primary Dependencies**: FastAPI, PyQt6 + PyQt6-WebEngine, Jinja2 (already in requirements), Pillow, moondream (optional pip install)  
**Storage**: Files only — Markdown + PDF written to `output_dir` (default: `Path.home() / "Desktop"`). Moondream model weights cached at `~/.cache/huggingface/` (HuggingFace default).  
**Testing**: pytest — TDD; `tests/test_intel_report.py` written first (12 tests, all fail before implementation)  
**Target Platform**: Windows 11 primary, cross-platform Python  
**Project Type**: Desktop app (PyQt6 + FastAPI single-binary-style server + SPA)  
**Performance Goals**: Report generation (Markdown + PDF) in under 2 minutes for sessions with up to 50 events, with Moondream2 model already cached or not installed  
**Constraints**: Markdown output < 100KB; all paths via `pathlib.Path`; offline-capable after first-run model download  
**Scale/Scope**: Single video, up to 60 minutes, up to 200 events, 1080p resolution

## Constitution Check

*GATE: Pass before Phase 0 research. Re-check after Phase 1 design.*

1. **Principle I (Session-First)**: Report generation writes Markdown + PDF to `output_dir` — these are export artifacts, not job state. No new session fields added. Moondream model cache uses HuggingFace's default location (`~/.cache/huggingface/`) — analogous to how `imageio-ffmpeg` uses its own cache location (not all model-caching uses `.cctv_processor`; the constitution's YOLO cache rule is YOLO-specific). **COMPLIANT.**

2. **Principle II (Cross-Platform)**: All file paths via `pathlib.Path`. No bare FFmpeg/ffprobe strings (FFmpeg not used in new code). Output folder defaults to `Path.home() / "Desktop"`. **COMPLIANT.**

3. **Principle III (Test-First)**: `tests/test_intel_report.py` contains 12 TDD tests covering all new `app/core/` modules. These MUST be written and run to confirm failure before any `app/core/` implementation is written. Frontend `export.js` changes use the frontend exemption — verified by the driving-script + manual quickstart scenarios documented in `quickstart.md`. **COMPLIANT.**

4. **Principle IV (Callback-Driven)**: No new detection or export engines. Intelligence report generation is a synchronous, single-request API call (not a long-running engine). `thumbnail_gen.run()` is called directly (already compliant). **COMPLIANT.**

5. **Principle V (YAGNI)**: Minimal new components, reusing Phase 5's exact Jinja2 + Qt printToPdf patterns. Moondream is an optional dependency with explicit ImportError fallback — the app functions fully without it. No new abstractions beyond what's required. **COMPLIANT.**

**Result: No violations. No Complexity Tracking table needed.**

## Global Constraints for Implementers

The following constraints MUST be followed across all tasks:

1. **TDD sequence for `app/core/`**: Write test → run `pytest tests/test_intel_report.py -v` to confirm failure → implement → run to confirm pass → commit.
2. **Paths**: Every file path MUST use `pathlib.Path`. No string concatenation with `/` or `\\`.
3. **Moondream optional**: `frame_describer.py` MUST handle `ImportError` (moondream not installed) and any `Exception` from `describe()` by returning `""` — no exception propagates to the caller.
4. **Output directory**: Resolve as `Path(snap.get("output_dir") or (Path.home() / "Desktop"))`, same as the CSV/JSON export endpoints.
5. **Endpoint guards** (both new endpoints):
   - Return 400 if no active job (`not snap.get("job_id")`)
   - Return 400 if status is `"detecting"`
   - Return 400 if no included events
6. **Filename pattern**: `{source_stem}_intelligence_{YYYYMMDD_HHMMSS}.md` / `.pdf`
7. **Thumbnail generation**: Call `thumbnail_gen.run(job_id, source_path, included, logger)` before base64-encoding thumbnails — it is idempotent and skips existing files.
8. **JSON appendix in Markdown**: Valid `json.loads()`-parseable. Each event object MUST contain: `event_index`, `start_s`, `end_s`, `start_clock`, `end_clock`, `peak_motion_score`, `zone_label`, `included`. Optional: `description`.
9. **Phase 5 PDF pattern**: `_generate_intel_report_pdf()` in `main_window.py` follows `_generate_pdf_report()` exactly — hidden `QWebEnginePage`, loads `/api/job/intel-report.html`, `printToPdf` on `loadFinished`, `deleteLater()` + list removal on `pdfPrintingFinished`.
10. **Frontend exemption**: `export.js` changes are NOT covered by pytest — they are verified by the driving script scenario in `quickstart.md`.
